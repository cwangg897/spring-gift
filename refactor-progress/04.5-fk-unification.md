## [필수 3] 목표 산출물 + 체크리스트 (단일 phase)

### Phase (구조 변경, JPA 매핑만)

- [ ] **12.1** `Wish.memberId: Long` → `Wish.member: Member @ManyToOne(fetch=LAZY)` 변경 — (a) `Wish.java:17` 매핑 교체, `@JoinColumn(name="member_id")` 명시 (b) `Wish(Long memberId, Product product)` 생성자 시그니처 → `Wish(Member member, Product product)` 로 변경, 호출처 (`WishController.java:74` 또는 service) 일괄 변경 (c) `./gradlew test` 그린
- [ ] **12.2** `Order.memberId: Long` → `Order.member: Member @ManyToOne(fetch=LAZY)` 변경 — (a)/(b)/(c) 12.1 동일 패턴, 호출처 (`OrderFacade.createOrder`) 일괄 변경
- [ ] **12.3** Repository 메서드 시그니처 정합 — (a) `WishRepository.findByMemberId(Long, Pageable)` → 유지(JPQL `where w.member.id = ?1`) 또는 `findByMember(Member, Pageable)` 로 변경. `OrderRepository.findByMemberId(Long, Pageable)` 동일 (b) 호출처 (`WishController.java:47`, `OrderController.java:57`) 갱신 (c) 페이지네이션 통합 테스트 그린
- [ ] **12.4** **N+1 정책 박제 (ADR-003)** — (a) 읽기 경로(`getWishes`, `getOrders`)에 `@EntityGraph(attributePaths = {"product", "member"})` 또는 JPQL `select new XxxResponse(...)` projection 적용 (b) Hibernate Statistics 로 쿼리 카운트 측정 + N+1 회귀 없음 검증 (c) 페이지 크기 N 일 때 SQL 실행 ≤ 2건 (count + select)
