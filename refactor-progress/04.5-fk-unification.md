# 04.5-fk-unification · JPA 매핑 통일 (횡단)

> **PR #12**. 횡단 단계, 단일 phase.
> **ADR-003 정정**: V1 스키마가 이미 `bigint FK` 보유 → **Flyway V3 마이그레이션 불필요**. JPA 매핑만 통일.

---

## 1. 현재 상태 진단

- 패키지: `gift.wish`, `gift.order`
- 대상 파일:
  - `Wish.java:17` — `private Long memberId;` (primitive FK)
  - `Wish.java:19-21` — `@ManyToOne Product product;` (이미 매핑)
  - `Order.java:25` — `private Long memberId;` (primitive FK)
  - `Order.java:21-23` — `@ManyToOne Option option;` (이미 매핑)
  - `V1__Initialize_project_tables.sql:31-32, 51` — `bigint not null` FK 정의 (스키마 OK)

---

## 2. 산출물 + 체크리스트 (단일 phase, JPA 매핑만)

- [x] 12.1 `Wish.memberId: Long` → `Wish.member: Member @ManyToOne(fetch=LAZY) @JoinColumn(name="member_id")`. 생성자 `(Member, Product)`, 호출처 일괄 갱신.
- [x] 12.2 `Order.memberId: Long` → `Order.member: Member @ManyToOne(fetch=LAZY) @JoinColumn(name="member_id")`. 생성자 `(Option, Member, int, String)`.
- [x] 12.3 Repository 시그니처: Spring Data 중첩 표기 `findByMember_Id` / `findByMember_IdAndProduct_Id`. 호출처 (`WishController`, `OrderController`) 갱신.
- [x] 12.4 N+1 방지 EntityGraph: `WishRepository.findByMember_Id` → `attributePaths={"product"}`, `OrderRepository.findByMember_Id` → `attributePaths={"option", "option.product"}`.
