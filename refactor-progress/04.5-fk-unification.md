# 04.5-fk-unification · JPA 매핑 통일 (횡단)

> **PR #12**. 횡단 단계, 단일 phase.
> **ADR-003 정정**: V1 스키마가 이미 `bigint FK` 보유 → **Flyway V3 마이그레이션 불필요**. JPA 매핑만 통일.

---

## 1. 현재 상태 진단

- 패키지: `gift.wish`, `gift.order`
- 대상 파일:
  - `Wish.java:17` — `private Long memberId;` (primitive FK)
  - `Wish.java:19-21` — `@ManyToOne Product product;` (이미 매핑)
  - `Order.java:25` — `private Long memberId;` (primitive FK)
  - `Order.java:21-23` — `@ManyToOne Option option;` (이미 매핑)
  - `V1__Initialize_project_tables.sql:31-32, 51` — `bigint not null` FK 정의 (스키마 OK)

---

## 2. 산출물 + 체크리스트 (단일 phase, JPA 매핑만)

- [ ] 12.1 `Wish.memberId: Long` → `Wish.member: Member @ManyToOne(fetch=LAZY)` 변경, `@JoinColumn(name="member_id")` 명시, 생성자 시그니처와 호출처 일괄 갱신
- [ ] 12.2 `Order.memberId: Long` → `Order.member: Member @ManyToOne(fetch=LAZY)` 변경 (동일 패턴)
- [ ] 12.3 Repository 메서드 시그니처 정합 (`findByMemberId` 유지 + JPQL, 또는 `findByMember`)
- [ ] 12.4 **N+1 정책 (ADR-003)**: 읽기 경로(`getWishes`, `getOrders`)에 `@EntityGraph` 또는 JPQL projection 적용, 쿼리 카운트 ≤ 2 검증
